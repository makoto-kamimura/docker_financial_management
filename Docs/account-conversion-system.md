# 勘定科目変換システム仕様

家庭モード ↔ 法人モードを切り替える際の勘定科目自動変換フロー、  
手動マッピング UI、および設定画面での変換履歴確認機能の設計をまとめます。

---

## 1. 変換フロー概要

```
モード切替トリガー
        │
        ▼
┌─────────────────────────────────┐
│  STEP 1: 自動変換判定（全科目）  │
└─────────────────────────────────┘
        │
   ┌────┴────────────────────┐
   ▼                         ▼
定義済み科目              ユーザー独自科目
(H-prefix)                (任意登録)
   │                         │
変換テーブルで即決        段階的マッチング
   │                         │
   │              ┌───────────┼──────────────┐
   │         高信頼度     低信頼度        変換不可
   │         (自動候補)  (手動マッピング)  (変換不可表示)
   │              │           │              │
   └──────────────┴───────────┴──────────────┘
                  │
                  ▼
┌──────────────────────────────────────────┐
│  STEP 2: 変換確認画面（全科目まとめて）   │
└──────────────────────────────────────────┘
                  │
            ユーザーが確認・修正
                  │
                  ▼
             変換を確定・反映
             （変換ログを保存）
```

---

## 2. 自動変換の判定ロジック

ユーザー独自科目は以下の順で判定し、最初にマッチした段階で候補を決定します。

| 優先度 | 判定方法 | 信頼度 | 例 |
| ------ | -------- | ------ | -- |
| 1 | **変換テーブル直引き**（定義済みH-prefix科目） | 確定 | H-3001 → 水道光熱費 |
| 2 | **キーワードルール**（「税」「保険」「代」等） | 高 | 「市民税」→ 租税公課 |
| 3 | **あいまい一致**（科目名の類似度スコア） | 中 | 「外食費」→ 接待交際費 |
| 4 | **AI推論**（LLMによる候補提示） | 低〜中 | 「推し活費」→ 娯楽費 |
| 5 | **手動マッピング必須** | — | 候補なし or 変換不可 |

信頼度スコアのしきい値（暫定）:

- **80% 以上** → 自動候補として確認画面に表示（変更可）
- **50〜79%** → 候補提示＋黄色ハイライトで要確認
- **50% 未満** → 手動マッピング画面へ誘導
- **変換不可フラグあり** → 変換不可として表示（ペット関連・理容費等）

## 3. 変換確認画面 仕様

### 表示項目

| 列 | 内容 |
| -- | ---- |
| 家庭モード科目名 | 変換元の科目名とコード |
| 変換先科目名 | 法人科目名とコード（変更可能） |
| 変換区分バッジ | 自動 / 要確認 / 手動 / 変換不可 |
| 操作 | 変換先を変更 / 手動マッピング画面を開く |

### バッジ定義

| バッジ | 色 | 意味 |
| ------ | -- | ---- |
| 自動変換 | 緑 | 信頼度80%以上で自動確定 |
| 要確認 | 黄 | 信頼度50〜79%、ユーザー確認推奨 |
| 手動設定 | 青 | ユーザーが手動で選択済み |
| 変換不可 | 赤 | 法人科目として存在しない |

### アクション

- **「変換先を変更」**: 法人科目一覧からドロップダウン選択
- **「確定して反映」**: 全科目の変換を一括適用し、変換ログを保存
- **「キャンセル」**: モード切替を中止し、現状を維持
- **「AIに詳細分析を依頼」**（有料）: 未確定・要確認科目をまとめてAI有料変換へ送信
- **「会計士に依頼」**（有料）: 変換結果一式を会計士レビュー依頼へ転送（Phase 後日）

### UX 原則

- 1件ずつではなく**一覧でまとめて表示 → まとめて確定**
- 変換不可科目は削除せず**家庭モードのデータとして保持**（モードを戻せば復元）
- ユーザーが手動で変更したマッピングは**次回以降の自動変換に学習として保存**

## 4. 設定画面：変換履歴確認

### 配置

```
設定 > 勘定科目 > 変換履歴
```

### 画面構成

```
┌────────────────────────────────────────────────────┐
│  勘定科目 変換履歴                                  │
│                                                    │
│  [2026-06-29] 家庭 → 法人  ▼                      │ ← 変換セッション選択
│                                                    │
│  ┌──────────────┬──────────────┬──────┬────────┐  │
│  │ 家庭モード科目│ 法人モード科目│変換区分│ 変更日 │  │
│  ├──────────────┼──────────────┼──────┼────────┤  │
│  │ 給与         │ 役員報酬     │ 自動  │ 06-29  │  │
│  │ 電気代       │ 水道光熱費   │ 自動  │ 06-29  │  │
│  │ 推し活費     │ 娯楽費       │ 手動  │ 06-29  │  │
│  │ ペット医療費  │ ─（変換不可）│変換不可│ 06-29  │  │
│  └──────────────┴──────────────┴──────┴────────┘  │
│                                                    │
│  [マッピングを修正]  [この変換セッションを再実行]   │
└────────────────────────────────────────────────────┘
```

### 機能

| 機能 | 説明 |
| ---- | ---- |
| **変換セッション一覧** | 過去のモード切替ごとに履歴を管理 |
| **詳細表示** | セッションを選択すると全科目の変換結果を一覧表示 |
| **マッピング修正** | 過去の変換結果を後から修正可能（次回切替時に自動適用） |
| **再実行** | 同じ設定でモード切替の変換を再実行 |
| **CSV エクスポート** | 変換履歴を CSV でダウンロード |

## 5. データ設計（概要）

### AccountMappingRule（変換ルールテーブル）

| カラム | 型 | 説明 |
| ------ | -- | ---- |
| `id` | Int | PK |
| `homeCode` | String | 家庭モード科目コード（H-prefix または任意） |
| `corporateCode` | String | 法人科目コード |
| `matchType` | Enum | TABLE / KEYWORD / FUZZY / AI_FREE / AI_PAID / MANUAL |
| `confidenceScore` | Float | 信頼度スコア（0.0〜1.0） |
| `isConvertible` | Boolean | 変換不可フラグ |
| `userId` | Int? | null=システム定義、値あり=ユーザー独自ルール |
| `createdAt` | DateTime | |
| `updatedAt` | DateTime | |

### AccountConversionSession（変換セッションテーブル）

| カラム | 型 | 説明 |
| ------ | -- | ---- |
| `id` | Int | PK |
| `userId` | Int | 実行ユーザー |
| `fromMode` | Enum | HOME / CORPORATE |
| `toMode` | Enum | HOME / CORPORATE |
| `convertedAt` | DateTime | 変換実行日時 |
| `status` | Enum | COMPLETED / CANCELLED |

### AccountConversionLog（変換ログテーブル）

| カラム | 型 | 説明 |
| ------ | -- | ---- |
| `id` | Int | PK |
| `sessionId` | Int | FK → AccountConversionSession |
| `homeAccountId` | Int | 変換元科目 |
| `corporateAccountId` | Int? | 変換先科目（変換不可の場合 null） |
| `matchType` | Enum | 変換方法 |
| `confidenceScore` | Float? | 信頼度スコア |
| `isConvertible` | Boolean | 変換可否 |
| `isManuallyOverridden` | Boolean | ユーザーが手動で変更したか |
| `aiResultId` | Int? | FK → AiConversionResult（AI有料変換の場合のみ） |

### AiConversionResult（AI有料変換結果キャッシュテーブル）

| カラム | 型 | 説明 |
| ------ | -- | ---- |
| `id` | Int | PK |
| `cacheKey` | String | ハッシュキー（accountName + businessType） |
| `inputContext` | Json | AIに渡したコンテキスト全体 |
| `primaryCode` | String | 推奨変換先科目コード |
| `primaryReason` | String | 選択理由（日本語テキスト） |
| `taxClassification` | String | 消費税区分（課税/非課税/免税/不課税） |
| `taxNotes` | String? | 税務上の注意事項 |
| `alternatives` | Json | 代替候補リスト（code/name/reason/tradeoff） |
| `confidenceScore` | Float | 信頼度スコア（0.0〜1.0） |
| `confidenceReason` | String | 信頼度の根拠説明 |
| `warnings` | Json | 注意事項リスト |
| `modelId` | String | 使用したLLMモデルID |
| `cachedUntil` | DateTime | キャッシュ有効期限（30日） |
| `createdAt` | DateTime | |

### AiConversionUsage（AI有料変換利用履歴テーブル）

| カラム | 型 | 説明 |
| ------ | -- | ---- |
| `id` | Int | PK |
| `userId` | Int | FK → User |
| `sessionId` | Int | FK → AccountConversionSession |
| `accountCount` | Int | 変換対象科目数 |
| `cachedCount` | Int | キャッシュヒット数（課金対象外） |
| `billedCount` | Int | 実際にAPI呼び出しした科目数 |
| `plan` | Enum | PAY_PER_USE / MONTHLY |
| `chargedAt` | DateTime | 課金日時 |
| `reportGenerated` | Boolean | PDF報告書を生成したか |

## 6. 実装フェーズ

| フェーズ | 内容 |
| -------- | ---- |
| **Phase 1** | DB設計・migration（AccountMappingRule / Session / Log） |
| **Phase 2** | 自動変換ロジック（キーワード→あいまい→AI無料推論の段階判定） |
| **Phase 3** | 変換確認画面（フロントエンド） |
| **Phase 4** | 設定画面「変換履歴」ページ（フロントエンド） |
| **Phase 5** | モード切替 API への変換フロー組み込み |
| **Phase 6** | AI有料変換ロジック・課金フロー・PDF報告書生成 |

---

## 7. 課金機能概要

変換確認画面に以下の有料オプションを追加します。  
会計士依頼は将来フェーズとし、まず AI 有料変換を先行実装します。

```
変換確認画面
  │
  ├─ [自分で確認して確定]         ← 無料（常時利用可）
  ├─ [AIに詳細分析を依頼] 💳      ← 有料（本セクション対象）
  └─ [会計士に依頼]       💳      ← 有料（将来フェーズ）
```

### 料金モデル（案）

| プラン | 価格 | 内容 |
| ------ | ---- | ---- |
| 都度課金 | ¥980 / 回 | モード切替1セッション分の AI 変換 |
| 月額プラン | ¥480 / 月 | 月内の AI 変換が無制限 |
| 初回無料 | — | 初回セッションのみ無料で体験可能 |

> 同一キャッシュキーにヒットした科目（30日以内）は課金対象外とし、API コストを削減します。

## 8. AI有料変換ロジック詳細

### 8-1. 無料 AI との差分

| 比較軸 | 無料 AI（Step 4） | 有料 AI |
| ------ | ----------------- | ------- |
| 入力情報 | 科目名のみ | 科目名 + 取引傾向 + 事業プロフィール + 関連科目一覧 |
| 使用モデル | 軽量モデル（高速・低コスト） | 高精度モデル（Claude Sonnet / Opus 等） |
| 推論方式 | シングルパス | 多段階 Chain-of-Thought |
| 出力 | 候補科目 + 信頼度スコア | 候補科目 + 根拠説明 + 代替案 + 消費税区分 + 税務注意事項 |
| 信頼度根拠 | スコアのみ | スコア + 根拠テキスト |
| 報告書 | なし | PDF変換報告書を生成（オプション） |
| キャッシュ | なし | 同一キャッシュキーで30日間再利用 |

### 8-2. 処理フロー

```
[有料AI変換リクエスト]
        │
        ▼
① キャッシュ確認
   cacheKey = hash(accountName + businessType)
   ├─ HITあり → キャッシュ結果を返却（課金なし）
   └─ HITなし → 次ステップへ
        │
        ▼
② コンテキスト収集
   ├─ 対象科目の取引履歴（直近12ヶ月）
   │    月平均金額・取引回数・季節変動 を集計
   ├─ ユーザーの事業プロフィール（業種・設立年等）
   └─ 同一ユーザーの他の勘定科目一覧（参考文脈）
        │
        ▼
③ 事前分類（ルールベース）
   ├─ 科目カテゴリ推定（収入 / 費用 / 資産 / 負債）
   ├─ 金額帯から固定費 / 変動費を推定
   └─ 個人支出 / 事業支出の仮判定
        │
        ▼
④ LLM呼び出し（高精度モデル）
   ├─ System prompt: 日本の税理士・会計士ペルソナ
   ├─ User prompt: 構造化コンテキスト（下記参照）
   └─ 出力形式: JSON（下記スキーマ参照）
        │
        ▼
⑤ 後処理・バリデーション
   ├─ 提案科目が法人科目マスタに存在するか確認
   ├─ 消費税区分の整合性チェック
   ├─ 信頼度スコアの最終調整
   └─ 人間確認フラグの付与（信頼度60%未満の場合）
        │
        ▼
⑥ 結果保存 & 返却
   ├─ AiConversionResult にキャッシュ保存
   ├─ AiConversionUsage に利用ログ保存
   └─ 変換確認画面へ結果を返却
```

### 8-3. プロンプト設計

**System prompt**
```
あなたは日本の税理士・公認会計士の知識を持つ財務AIアドバイザーです。
日本の企業会計基準・法人税法・消費税法に基づき、
家庭モードの勘定科目を法人会計の勘定科目へ変換してください。
回答は必ず指定のJSONフォーマットで返してください。
```

**User prompt（構造化コンテキスト）**
```
【変換対象科目】
科目名: {accountName}
カテゴリ: {category}
月平均取引額: {avgMonthlyAmount}円
月平均取引回数: {avgMonthlyCount}回
季節変動: {seasonalPattern}

【ユーザー事業情報】
業種: {businessType}
設立・開業年: {establishedYear}

【関連科目（同一ユーザーの他科目・参考）】
{relatedAccountsList}

【変換候補（法人科目マスタ）】
{corporateAccountMasterExcerpt}
```

**出力JSONスキーマ**
```json
{
  "primary": {
    "code": "法人科目コード",
    "name": "法人科目名",
    "reason": "選択理由（2〜3文）",
    "taxClassification": "課税 | 非課税 | 免税 | 不課税",
    "taxNotes": "税務上の注意事項（任意）"
  },
  "alternatives": [
    {
      "code": "...",
      "name": "...",
      "reason": "...",
      "tradeoff": "推奨候補との違い・使い分け"
    }
  ],
  "confidence": {
    "score": 0.85,
    "reasoning": "信頼度の根拠（例: 金額帯・取引頻度が典型的な○○費のパターンに合致）"
  },
  "warnings": [
    "按分計算が必要な可能性があります（私用兼業の場合）"
  ]
}
```

### 8-4. 信頼度スコアと確認画面の対応

| スコア | バッジ | 色 | 確認画面の扱い |
| ------ | ------ | -- | -------------- |
| 0.85以上 | AI自動 | 緑 | そのまま確定候補として表示 |
| 0.60〜0.84 | AI要確認 | 黄 | 候補表示 + ユーザー確認推奨 |
| 0.60未満 | AI低信頼 | オレンジ | 候補表示 + 代替案を並列表示 + 手動変更を促す |

### 8-5. キャッシュ戦略

```
cacheKey = SHA256(accountName + ":" + businessType)

キャッシュ有効期限: 30日
無効化タイミング:
  - ユーザーが事業プロフィール（業種等）を更新したとき
  - 法人科目マスタが更新されたとき
  - ユーザーが「再分析」を明示的に実行したとき

キャッシュヒット時: 課金なし・即時返却
キャッシュミス時: LLM API呼び出し → 結果保存 → 課金
```

### 8-6. PDF変換報告書（オプション）

変換確認後、「変換報告書をダウンロード」ボタンから生成可能。

| セクション | 内容 |
| ---------- | ---- |
| 表紙 | 変換実行日・ユーザー名・変換方向（家庭→法人） |
| 変換一覧 | 全科目の変換元・変換先・変換区分・信頼度 |
| AI分析結果 | AI有料変換科目の根拠説明・消費税区分・税務注意事項 |
| 変換不可一覧 | 変換不可科目の一覧と理由 |
| 注意事項 | 本報告書は参考情報であり、税務申告の根拠とはならない旨の免責事項 |
